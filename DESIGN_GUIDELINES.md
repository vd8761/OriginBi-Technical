# OriginBI Design & Development Guidelines
## Eliminating "AI Slop Design" & Maintaining Premium Product Fidelity

These guidelines are a strict ruleset for developers, designers, and AI coding assistants working on the OriginBI ecosystem. They outline the common symptoms of "AI Slop Design" (unnecessary visual clutter and over-styled components generated by automated models) and prescribe exact standardizations to maintain a clean, performant, and high-fidelity SaaS interface.

---

## 1. What is "AI Slop Design"?

AI Slop Design refers to the automatic injection of heavy, complex, or trendy visual effects that degrade usability, decrease application performance, and create massive inconsistency. 

### Core Symptoms to Avoid:
*   **Excessive Glowing Shadows**: Adding colored neon shadows (e.g., `shadow-brand-green/30` or `shadow-[0_0_15px_...]`) behind standard menu buttons, logos, or icons.
*   **Over-Animated Interfaces**: Saturating simple hover actions with heavy bounce/spring logic, rotational animations on avatars, or infinite pulse loops on standard status indicators.
*   **Extreme Rounded Corners**: Using `rounded-3xl` or `rounded-[32px]` for simple inputs, dropdown lists, or small badges.
*   **Excessive Gradients**: Using multi-color neon gradients (e.g., green-to-blue-to-pink) on normal headers, buttons, or texts.
*   **Placeholder Graphics & Avatars**: Inserting heavy mock icon graphics instead of standard user-centric components like Initials Avatars.

## 2. Core Styling & Aesthetics Standards

### Color Harmony
*   **Primary Brand Green**: `#1ED36A` (represented as `--color-brand-green` or `bg-brand-green`).
*   **Dark Neutral Background**: `#19211C` (`dark:bg-brand-dark-primary`).
*   **Card Backgrounds**: 
    *   Light mode cards must use crisp white or grey borders: `bg-white border border-gray-150`.
    *   Dark mode cards should use muted dark grey backgrounds with thin borders: `bg-[#24272B]/90 border border-white/[0.08]`.
*   **No Neon Accents**: Avoid highlighting non-actionable elements with purple, pink, or cyan highlights. Keep the branding cohesive with Greens, Slate-Greys, and Deep Navy accents.

### Typography
*   **Font-Family**: The entire interface is standardly rendered in **Haskoy** for high-end SaaS readability.
*   **Weights & Sizing**:
    *   Header title text should be `font-semibold` or `font-bold` with proper tracking-tight classes.
    *   Subtext and descriptions must be `text-gray-500` or `dark:text-gray-400` with clean, standard body line-height.
    *   Do not overlay text on top of complex backgrounds without a semi-opaque backdrop blur panel.

### Layout & Border-Radius
*   **Buttons & Pills**: Use `rounded-full` for menu buttons and badges.
*   **Cards**: Standard cards should use `rounded-xl` (12px) or `rounded-2xl` (16px). Avoid using `rounded-3xl` (24px) unless it's a massive, focal dashboard module.
*   **Dropdowns**: Standard profile lists or settings menus must be `rounded-xl` (12px) to look sharp and cohesive.

---

## 3. Micro-Interactions & Performance

*   **Standard Transitions**: Use standard vanilla CSS transitions (`transition-all duration-200 ease-out`) rather than importing complex scroll listeners or framer-motion values for simple components.
*   **Framer-Motion Limit**: Frame-by-frame spring curves look sluggish and introduce UI lag on heavy tables. Limit the use of Framer Motion to screen-wide overlay modals or complex layout changes. Never use it on button hover or profile avatar triggers.
*   **Lightweight Navigation**: The header background should rely on high-fidelity CSS backdrops:
    `bg-white/95 dark:bg-[#19211C]/95 backdrop-blur-xl`
    Avoid monitoring window scroll coordinates in React state to toggle header opacity unless absolutely necessary.

## 4. Specific Component Standards

| Component | AI Slop (DO NOT DO) | Premium Clean (MUST DO) |
| :--- | :--- | :--- |
| **Active Nav Item** | Green gradient backdrop + glowing colored shadow + scale-up hover. | Flat solid background (`bg-brand-green text-white`) + zero shadow. |
| **User Profile Section** | mock SVG profile icon + rotating borders + "Technical Portal" sub-text. | Initials-based dynamic image (`ui-avatars.com`) + user email subtitle + clean click dropdown. |
| **Notifications Dropdown** | Tiny card (`380px`) + fancy popup scale animations + glowing indicators. | Wide card (`480px-540px`) + standard header + tab buttons + clean `divide-y` list border separators. |
| **Glow Effects** | Glow panels behind logo or titles using deep blur drop-shadows. | Transparent elements with zero blur-background drop shadow. |

---

## 5. Summary Checklists for Developers

Before merging or approving any pull request:
1.  **Check Shadows**: Are there any colored glows on simple buttons? If yes, remove them.
2.  **Check Animations**: Does the header or nav bar use framer-motion trigger states? Replace with crisp CSS transitions.
3.  **Check Layout Density**: Are card widths, paddings, and font sizes matching across portals? Ensure responsive styling (`sm:`, `md:`, `lg:`) is consistent.
4.  **Confirm Identity**: Does the active portal represent the main app's premium standard? Ensure logo sizes, font face values, and colors are exact duplicates.
